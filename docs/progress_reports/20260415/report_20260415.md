## Key Discovery: Toolbar Item Names Override Button Names

**Important:** Toolbar items have their own `name` field that **overrides** the button's display name in the UI.

```
Button entity:       name = "Сохранить"
Toolbar item:       name = "Save" (this takes precedence in UI!)
```

**How to change button label in toolbar:**
```python
# WRONG: Editing button doesn't affect toolbar display
edit_or_create_button.invoke({
    "button_system_name": "create",
    "name": "Новая запись"
})

# CORRECT: Update toolbar items array
edit_or_create_toolbar.invoke({
    "toolbar_system_name": "defaultListToolbar",
    "items": [
        {"button_system_name": "create", "display_name": "Новая запись"}
    ]
})
```
